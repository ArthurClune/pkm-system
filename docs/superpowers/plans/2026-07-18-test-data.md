# Synthetic Test and Development Data Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a small committed synthetic dataset that selectively supports automated tests and reproducibly generates the ignored local-development graph.

**Architecture:** A Functional Core parses and validates `test-data/graph.json`, expands named asset placeholders, and adapts the source into the importer's existing row model. A thin Imperative Shell reads files, hashes assets, builds SQLite in a staging directory with the current schema, and atomically publishes it to `data/` without silently overwriting a graph. Existing focused fixtures and temporary E2E databases remain unchanged.

**Tech Stack:** Python 3.12, Pydantic 2, SQLite/FTS5, pytest, existing PKM importer/schema APIs, TypeScript, Playwright, pnpm.

## Global Constraints

- Copy no text, database rows, or media from any prior local graph or `/tmp/graph-data`.
- Add no provenance/history document. `test-data/README.md` explains only current fixture coverage and usage.
- Commit source fixtures under root `test-data/`; keep generated `data/` ignored.
- Use valid renderable SVG and PDF fixtures, not placeholder bytes.
- Keep `server/tests/conftest.py`, importer-specific fixtures, in-memory web fixtures, and `server/tests/e2e_serve.py` isolated and unchanged.
- Use the shared root fixtures only in tests where they are the subject of the test.
- Preserve the existing asset contract: bytes at `data/assets/<sha[:2]>/<sha>` and graph URLs at `/assets/<sha>/<quoted-filename>`.
- Never overwrite an existing database or asset directory implicitly; preserve a lone existing `config.json` byte-for-byte.
- “Deterministic” means stable source, graph rows, hashes, and URLs. SQLite bytes need not match because `init_db()` creates a random sync-generation token.
- Every new runtime Python file must declare `# pattern: Functional Core` or `# pattern: Imperative Shell` near the top.
- Add no new package dependency.
- Use `uv`, never `pip`; type all Python APIs and add docstrings to public APIs.
- Run focused pytest cycles with `--no-cov`; run the project’s enforced coverage suite before completion.

```python
from __future__ import annotations

import re
from collections import defaultdict
from collections.abc import Set
from typing import Literal

from pydantic import BaseModel, ConfigDict, Field, StrictBool, StrictInt, StrictStr, ValidationError

ASSET_PLACEHOLDER_RE = re.compile(r"\{\{asset:([^{}]+)\}\}")


class SourceValidationError(ValueError):
    """Raised when the committed graph source violates its schema or invariants."""


class BlockSource(BaseModel):
    """One strictly typed block in the human-readable source graph."""

    model_config = ConfigDict(extra="forbid", frozen=True)

    uid: StrictStr = Field(min_length=1)
    parent_uid: StrictStr | None
    order_idx: StrictInt = Field(ge=0)
    text: StrictStr
    heading: Literal[1, 2, 3] | None
    collapsed: StrictBool
    view_type: Literal["numbered", "document"] | None
    created_at: StrictInt | None
    updated_at: StrictInt | None


class PageSource(BaseModel):
    """One page and its flat, relationship-validated source blocks."""

    model_config = ConfigDict(extra="forbid", frozen=True)

    title: StrictStr = Field(min_length=1)
    created_at: StrictInt | None
    updated_at: StrictInt | None
    blocks: tuple[BlockSource, ...]


class GraphSource(BaseModel):
    """Validated source graph and ordered sidebar titles."""

    model_config = ConfigDict(extra="forbid", frozen=True)

    pages: tuple[PageSource, ...]
    sidebar_entries: tuple[StrictStr, ...]
```

Implement `parse_graph_source(raw: object, *, asset_names: Set[str]) -> GraphSource` with a public docstring so it:

1. converts `ValidationError.errors()` locations into dotted paths such as `pages.0.blocks.0.heading` and raises `SourceValidationError`;
2. rejects duplicate page titles and globally duplicate block UIDs;
3. distinguishes unknown parents from parents owned by another page;
4. detects cycles by walking each block’s parent chain;
5. groups blocks by `parent_uid` and requires each sibling group’s sorted order indexes to equal `list(range(len(group)))`;
6. rejects duplicate sidebar titles and sidebar titles absent from explicit pages; and
7. finds every `ASSET_PLACEHOLDER_RE` match and rejects names absent from `asset_names`.

Keep all filesystem checks out of this module. Error strings must exactly match the tests above so CLI errors remain actionable.

from collections.abc import Mapping
from dataclasses import dataclass
from urllib.parse import quote

from pkm.importer.assets import Asset
from pkm.importer.parse_export import Block, Export, Page
from pkm.importer.rows import Rows, to_rows


@dataclass(frozen=True)
class PreparedGraph:
    """Database and sidebar rows derived from one validated source graph."""

    rows: Rows
    sidebar_rows: tuple[tuple[str, int], ...]


def expand_asset_placeholders(
    text: str,
    assets_by_name: Mapping[str, Asset],
) -> str:
    """Replace validated named fixtures with application asset URLs."""
    return ASSET_PLACEHOLDER_RE.sub(
        lambda match: (
            f"/assets/{assets_by_name[match.group(1)].sha256}/"
            f"{quote(assets_by_name[match.group(1)].filename)}"
        ),
        text,
    )
